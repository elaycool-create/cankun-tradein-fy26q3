# 燦坤 Trade-in 舊換新週報分析

**Description:** 分析 Apple Trade-in 舊換新原始 Excel 資料，依燦坤門市與週次（FY 週次）統計執行數、接機數（Plugin 🔌）、成交數，產出含 iPhone 銷量輸入、回收率即時計算、操作人排行、互動式搜尋的單頁 HTML 報告。

---

## 適用情境（When to use）

- 收到 Apple ITLS 系統匯出的 Trade-in Excel（典型欄位 56–58 欄、含 `订单号 / 经销商名称 / 门店名称 / 回收类型 / 检测信息 / XPOS操作人 / 订单创建时间`）
- 想以 Apple 財年週次（每週週日起始）追蹤門市表現
- 需要一份能上傳 GitHub Pages 公開分享、或單純桌面瀏覽的互動報告

---

## 分析定義（Metric definitions）

| 指標 | 公式 / 判斷邏輯 | 來源欄位 | 達標標準 |
|------|----------------|----------|---------|
| **執行數** | 該門市於該週的總筆數 | `订单号` | — |
| **接機數 / Plugin 🔌** | `检测信息` 中含有 IMEI 碼（15 位數字 `IMEI:\d{15}`） | `检测信息` | — |
| **成交數** | `回收类型 == "以旧换新"` | `回收类型` | — |
| **操作人** | `XPOS 操作人` 欄有實值（排除 null 替代字 `58`、`60`） | `XPOS操作人` | — |
| **回收率** | 成交數 ÷ iPhone 銷量 × 100% | 前端互動 | **≥ 10%** ✅ |
| **接線率** | 接機數 ÷ iPhone 銷量 × 100% | 前端互動 | **≥ 15%** ✅ |

> ⚠️ **null 替代值**：Apple 各次匯出的檔案會用不同數字（`58`、`60`…）表示空值。腳本會自動偵測。

> 🎯 **修改達標標準**：在 `analyze_and_report.py` 頂部編輯：
> ```python
> RECYCLE_TARGET = 10.0   # 回收率
> PLUGIN_TARGET  = 15.0   # 接線率
> ```

## FY 週次定義

以 **2026/3/29（週日）** 為 FY26Q3W01 起點，每 7 天一週：

| 週次 | 日期範圍 |
|------|----------|
| FY26Q3W01 | 3/29 – 4/4 |
| FY26Q3W02 | 4/5 – 4/11 |
| FY26Q3W03 | 4/12 – 4/18 |
| FY26Q3W04 | 4/19 – 4/25 |
| FY26Q3W05 | 4/26 – 5/2 |
| FY26Q3W06 | 5/3 – 5/9 |
| FY26Q3W07 | 5/10 – 5/16 |
| … | 之後每 7 天累加 |

如需調整起始週或新增週次，編輯 `analyze_and_report.py` 中的 `WEEK_START` 常數即可。

## 報告內容（HTML 區塊）

1. **頂部 Header** — 期間、門市數、總執行筆數、操作人數
2. **KPI 卡片** — 總執行 / 接機 / 成交 / 操作人數（+ 最新週次 KPI）
3. **週次總覽表** — 每週執行 / Plugin / 成交
4. **操作人成交總排行** — 🥇🥈🥉 + 主要服務門市
5. **熱門舊機 / 換購新品 Top 10**
6. **iPhone 回收率／接線率排行榜** — 自動依使用者輸入更新（localStorage 儲存），依**回收率**降冪排序，達標門市顯示 ✅
7. **各門市卡片** —
   - KPI chips：執行 / Plugin / 成交
   - **iPhone 銷量輸入欄 → 回收率 + 接線率**（雙 badge 即時計算）
     - 達標 ≥ 標準 → 綠色 ✅
     - 接近（≥ 標準的一半）→ 橙色
     - 待加強 → 紅色
   - 全期操作人 pill
   - 7 週逐週明細（含舊機、換購新品 Top 3、操作人）

## 使用方式

### 1. 安裝相依套件

```bash
pip3 install pandas openpyxl
```

### 2. 執行（單檔分析）

```bash
python3 analyze_and_report.py \
  --input  "/path/to/原始資料.xlsx" \
  --output "/path/to/燦坤週報.html"
```

### 3. 增量更新（合併多個 Excel）

如果新一週的資料是「只含新一週」的 Excel，可以一次傳多個輸入：

```bash
python3 analyze_and_report.py \
  --input "舊資料W01-W06.xlsx" "新資料W07.xlsx" \
  --output "燦坤週報_W01-W07.html"
```

### 4. 自訂選項

| 參數 | 說明 | 預設 |
|------|------|------|
| `--input` | 一個或多個 Excel 路徑（可累加合併） | **必填** |
| `--output` | 輸出 HTML 路徑 | `./燦坤週報.html` |
| `--dealer` | 過濾的經銷商名稱關鍵字 | `燦坤` |
| `--start-date` | FYxxQxW01 起始日（YYYY-MM-DD） | `2026-03-29` |
| `--num-weeks` | 要展示的週數 | `自動推算` |
| `--export-json` | 同時匯出處理後的 JSON（供下次增量用） | 不輸出 |

## 修改 / 擴充建議

- **新增門市**：腳本會自動偵測 `门店名称` 中所有出現過的門市
- **加入其他經銷商**：把 `--dealer` 改為 `德誼` / `晶實` / `立展` 即可重複使用
- **細到每日**：把 `WEEKS` 邏輯改為依 `订单创建时间.dt.date` 分組
- **匯出 CSV**：在 `main()` 結尾加 `df.to_csv()` 即可

## 獎金規則（v7）

| 條件 | 高回收(≥$10,001) | 低回收(≤$10,000) |
|------|------------------|------------------|
| **基本**（月總 < 20萬） | 總額 × 1% | 每件 $100 |
| **進階**（月總 ≥ 20萬） | 總額 × 2% | 每件 $200 |

> 計算範圍：每月、每位操作人各別計算（依 `XPOS操作人` 欄）；以**訂單創建時間**的年-月為月份依據。
> 修改門檻：在 `analyze_and_report.py` `<script>` 段中編輯 `HIGH_THRESHOLD` / `ADVANCE_THRESHOLD` 等常數。
